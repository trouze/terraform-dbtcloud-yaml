# DPS E2E Import Profile

This folder is an isolated E2E “profile” so you can run fetch → normalize → Terraform plan/apply against a separate system/account (DPS) without overwriting your default `test/e2e_test` workspace.

## Files

- `env.example`: template for `./.env` (DPS credentials)
- `importer_mapping.yml`: DPS starter mapping config
- `main.tf`: Terraform root for the DPS test workspace
- `dbt-cloud-config.yml`: generated by the E2E script (do not hand-edit)

## Setup

1. Copy the env template:
   - `cp test/e2e_dps/env.example test/e2e_dps/.env`
2. Edit `test/e2e_dps/.env` and set at minimum:
   - `DBT_SOURCE_HOST_URL`
   - `DBT_SOURCE_ACCOUNT_ID`
   - `DBT_SOURCE_API_TOKEN`
   - `DBT_TARGET_HOST_URL`
   - `DBT_TARGET_ACCOUNT_ID`
   - `DBT_TARGET_API_TOKEN`
3. If you have GitHub App repos or GitLab OAuth repos in scope, set:
   - `DBT_TARGET_PAT` (a `dbtu_...` PAT)

## Run

From repo root:

- Plan-only:
  - `./test/run_e2e_test.sh --profile dps`
- Apply:
  - `./test/run_e2e_test.sh --profile dps --apply`
- Destroy:
  - `./test/run_e2e_test.sh --profile dps --destroy`

## Output locations

- Terraform workspace: `test/e2e_dps/`
- Exports/normalized artifacts: `dev_support/samples_dps/`
